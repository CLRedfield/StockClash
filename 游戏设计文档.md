# 《操盘杀 · StockClash》游戏设计文档

> 一局四分钟，跑完现实里四个月的行情。看谁是股神，谁在被割韭菜。
>
> 多人在线 / 单机对 AI 的「实时操盘竞速」游戏：随机抽一只真实标的、从随机历史时刻开局，
> 所有人同样的起始资金，逐秒跳动的真实 K 线，做多做空、贷款加杠杆，时间到结算，比谁钱最多。

## 1. 一句话定位

**「现实股市的时间压缩竞速」**：把真实历史行情按比例加速回放（默认一局 4 分钟 ≈ 现实 4 个月，每 1 秒刷新一次价格），
玩家在同一段行情里同台操盘，靠**择时、做多做空、杠杆贷款**拉开净值差距，结算时谁的总资产最多谁赢。

核心爽点：
- **真实**：用真实历史 K 线，不是凭空捏造的随机数。涨跌有"道理"，复盘有谈资。
- **快**：4 分钟一局，节奏紧凑，适合开黑连打。
- **博弈**：贷款利滚利、做空爆仓、道具搅局——既考验眼光也考验心态（梭哈还是落袋）。

## 2. 玩法总览

### 2.1 一局的生命周期

```
大厅 → 房间(配置/座位管理) → 开局倒计时 → 实时交易(默认4分钟) → 结算排名 → 再来一局/回房间
```

1. **大厅 Home**：三个入口——`本地创建房间` / `联机创建房间` / `加入房间`。
2. **房间 Room（等待区）**：房主配置规则、管理座位（踢人 / 换位 / 增减 AI / 调 AI 难度）；其他玩家只读，看房主的配置。
3. **开局**：房主点「开始」→ 房主随机抽取标的与起始时刻 → 3 秒倒计时 → 行情开始逐秒推送。
4. **交易阶段**：所有人面对同一条实时 K 线，自由买入 / 卖出 / 做空 / 平仓 / 贷款；右侧实时排行榜显示所有人净值。开启道具模式则每隔一段时间发放道具。
5. **结算**：时间到 → 强制按最后价格平掉所有持仓（含手续费）→ 扣除贷款本息 → 按最终净值排名 → 展示成绩与全程走势复盘。
6. **再来一局**：房主可回到房间（沿用 hearthStoneKill2 的"再来一局回房间"逻辑），重新配置或直接再开。

### 2.2 胜负判定

```
最终净值 = 现金 + Σ多头持仓市值 − Σ空头平仓所需金额 − 未偿贷款(本金+已计提利息)
```

结算时所有持仓按**最后一个 tick 的价格**强制平仓并扣手续费，再扣贷款本息，得到最终现金。**净值最高者获胜**，并列时比较"最大回撤更小者"。

## 3. 行情系统（核心）

### 3.1 时间缩放模型

| 参数 | 默认 | 说明 |
| --- | --- | --- |
| `gameDurationSec` 局时长 | `240`（4 分钟） | 一局的真实游戏时长 |
| `tickIntervalSec` 刷新间隔 | `1`（每秒） | 价格刷新一次 |
| `realWindow` 行情窗口 | `4 个月` | 这一局覆盖多长的真实历史行情 |

由此推导：

```
总 tick 数  totalTicks = gameDurationSec / tickIntervalSec       // 默认 240 个 tick
每 tick 推进的真实时间 = realWindow / totalTicks                  // 4个月/240 ≈ 0.5 天/tick
```

**预设档位**（房主可选，也可自定义）：

| 档位 | 局时长 | 行情窗口 | 刷新 | 体感 |
| --- | --- | --- | --- | --- |
| 闪电 | 2 min | 2 个月 | 1 s | 极快，适合热身 |
| 标准 ★ | 4 min | 4 个月 | 1 s | 默认 |
| 长局 | 8 min | 1 年 | 1 s | 能跑出大趋势 |
| 自定义 | 1–15 min | 1 周–3 年 | 0.5–2 s | 任意组合 |

### 3.2 真实数据 → 逐秒平滑

- **数据底座**：每只标的打包**真实历史日 K**（OHLCV）。4 个月 ≈ 84 个交易日（A股/港股/美股）/ ≈120 天（虚拟货币 7×24），而一局有 240 个 tick——tick 比日 K 更密。
- **插值 + 噪声**：以真实日 K 为锚点，相邻锚点之间做平滑插值，再叠加一层**按该资产历史波动率标定的布朗噪声**，得到逐秒跳动的价格——既忠于真实大方向，又有真实的逐秒手感（不会是一条折线）。
- **更细数据可选**：若某市场有 60 分钟 / 15 分钟 K，可直接回放，减少插值。
- **可复现**：行情由 `(标的ID, 起始索引, 随机种子)` 三元组完全决定，房主定好后所有端算出**完全一致**的价格序列（详见 §6 反作弊）。

### 3.3 随机开局

开局时房主从**已勾选的市场池**里：
1. 随机抽一只标的；
2. 在其历史里随机挑一个起始交易日（保证后面有足够长度覆盖 `realWindow`）；
3. 生成随机种子。

> **盲盒模式（可选开关）**：隐藏标的真实名称与日期，只显示代号（如「标的 A」），防止熟手凭记忆"开天眼"，也更公平刺激。

### 3.4 K 线来源（多选）

房间内勾选，可多选，开局从并集里随机抽：

| 来源 | 标识 | 交易时段特征 | 备注 |
| --- | --- | --- | --- |
| 大 A | `cn` | 工作日有涨跌停（±10%/±20%） | 波动相对温和，有涨跌停板规则 |
| 港股 | `hk` | 工作日，无涨跌停 | 波动大、仙股多 |
| 美股 | `us` | 工作日，无涨跌停 | 龙头白马 + 妖股 |
| 虚拟货币 | `crypto` | 7×24 不停 | 波动最猛，适合做空/杠杆党 |

每个市场内置一个**精选标的池**（数十只，含若干年历史），保证随机性与重玩性。

## 4. 经济系统

### 4.1 起始资金

`initialCash` 默认 `¥100,000`（房主可调）。所有人相同。

### 4.2 交易：做多 / 做空

| 操作 | 含义 | 盈亏 |
| --- | --- | --- |
| **买入（开多）** | 用现金买入 N 股 | 价格上涨获利 |
| **卖出（平多）** | 卖出持有的多头 | 锁定盈亏 |
| **做空（开空）** | 借券按现价卖出，记空头仓位 + 占用保证金 | 价格下跌获利 |
| **平空（买回）** | 买回平掉空头 | `(开空价 − 平仓价) × 数量 − 手续费` |

- 支持**部分平仓**与**一键全平**。
- 数量输入支持「按金额 / 按股数 / 全仓 25% / 50% / 100%」快捷档。

### 4.3 手续费（务必做进去）

每笔买卖都扣手续费。内置各市场近似费率档（**仅为游戏平衡，可在房间内统一调**）：

| 市场 | 买入费率 | 卖出费率 | 说明 |
| --- | --- | --- | --- |
| 大 A | 0.03% | 0.08% | 含佣金 + 卖出印花税近似 |
| 港股 | 0.13% | 0.13% | 含佣金 + 印花税 + 交易费近似 |
| 美股 | 0.01% | 0.02% | 近似零佣金 + 极小监管费 |
| 虚拟货币 | 0.10% | 0.10% | taker 费近似 |

- 可选**最低手续费**（如 ¥5/笔），抑制无脑高频。
- 手续费在每次成交时即时扣减，结算强平时同样收取。

### 4.4 贷款（杠杆）

让玩家能"借钱梭哈"，但利滚利、风险自负。

| 参数 | 默认 | 说明 |
| --- | --- | --- |
| 放款即计利 `originationRate` | +2% | **刚贷出**时立即在本金上加一次利息 |
| 周期复利 `accrualRate` | +2% | **每 30 秒**对未偿债务再复利一次 |
| 复利间隔 `accrualIntervalSec` | 30 s | 可调 |
| 杠杆上限 `maxLeverage` | 当前净值的 2× | 可借总额上限 |

债务增长（按"现实里百分比增加"）：

```
放款瞬间：  debt = principal × (1 + originationRate)
之后每 30s：debt = debt × (1 + accrualRate)        // 复利
```

规则要点（按需求）：
- **欠债时不能再贷款**——必须先还清当前债务才能发起新的贷款。
- 利息**不每秒扣现金**，而是累计在债务上，**结算时连本带息一并扣除**（也可中途主动还款减少利滚利）。
- 借来的钱进入可用现金，可用于开多 / 开空 / 补保证金。

### 4.5 强平 / 爆仓（维持游戏不崩）

- **维持保证金率** `maintenanceMargin` 默认 25%。空头或高杠杆持仓使保证金不足时触发**强制平仓（爆仓）**。
- 若**净值 ≤ 0** → 判定**破产出局**，锁定操作，以当时（负）净值参与排名垫底；其余玩家继续。
- 强平有醒目动画与音效提示（见 §7 / `fx`）。

### 4.6 结算

时间到（或全员出局）：
1. 按最后价格强制平掉所有多头与空头（收手续费）；
2. 扣除未偿贷款本息；
3. 计算最终净值并排名；
4. 展示：最终排名、收益率、最大回撤、交易次数、全程净值曲线 + 行情复盘（盲盒模式此时揭晓真实标的与日期）。

## 5. 房间与多人系统

### 5.1 两种创建方式

- **本地创建房间（单机）**：纯本地，1 名真人 + 若干 AI，无需联网。复用统一房间视图。
- **联机创建房间**：走公共 MQTT，**服务器方式完全沿用 hearthStoneKill2**（见 §6）。

### 5.2 房主权限（与 hearthStoneKill2 对齐）

房主在房间里可以：

| 权限 | 行为 |
| --- | --- |
| **踢人** | 把某真人移出房间（座位旁 ✕） |
| **换位 / 互换座位** | 点两个座位互换，或拖拽互换；可开关「允许玩家自由申请换座」 |
| **增 / 减 AI** | 空座位 = AI 补位；调整人数即增减 AI |
| **调 AI 难度** | 每个 AI 座位单独点按切换 简 / 普 / 难 |
| **规则配置** | K 线来源(多选)、档位/时长/窗口/刷新、初始资金、手续费率、贷款利率与间隔、盲盒、道具模式与间隔 |
| **开始 / 再来一局** | 开局；局后回到房间重配 |

> 这些配置只有房主可改，其他玩家端为只读（锁态），与 hearthStoneKill2 的 `canEdit / canKick / canSwap` 一致。

### 5.3 AI 操盘手

- AI 在**房主端**运行（房主权威），与真人一样**只能看到已揭示的历史 tick，不能预知未来**（公平）。
- 难度沿用 hearthStoneKill2 的"失误率"思路 `AI_CHAOS = { easy: 0.62, normal: 0.30, hard: 0.05 }`：

| 难度 | 风格 | 失误率 | 反应 |
| --- | --- | --- | --- |
| 简单 | 随机 + 追涨杀跌 | 高 | 慢、爱割肉 |
| 普通 | 趋势跟随 + 适度风控 | 中 | 均衡 |
| 困难 | 趋势 + 均值回归 + 风控 + 偶尔用杠杆/做空 | 低 | 快、稳 |

- AI 也会用贷款、做空、道具（道具模式下），让对局有"对手感"。

## 6. 联机架构（与 hearthStoneKill2 一致）

> 直接复用 hearthStoneKill2 的网络层：**免费公共 MQTT broker + 房主权威（host-authoritative）模式**。
> 无需自建服务器、无需注册、无需备案。可直接搬运 `src/net/mqtt.js` 的 `MqttBus`。

### 6.1 传输层

- 默认 broker：`wss://broker.emqx.io:8084/mqtt`，界面可切备用（`broker-cn.emqx.io`、`broker.hivemq.com` 等）。
- 6 位房间号（`genRoomCode`），主题命名空间隔离（如 `NS = 'stockclash1'`）。
- `MqttBus`：`connect / sub / pub / clearRetained`，QoS 1 关键消息、QoS 0 高频行情，retained 用于房间信息与快照。

### 6.2 主题表（在 hearthStoneKill2 基础上增改）

命名空间 `stockclash1/<房间号>/...`：

| 主题 | 方向 | retain | 说明 |
| --- | --- | --- | --- |
| `lobby` | 房主→全体 | ✓ | 房间配置 + 玩家/座位/AI/规则（房主权威发布） |
| `join` | 玩家→房主 | | 加入请求 |
| `move` | 玩家→房主 | | 换座申请（开启自由换座时） |
| **`feed`** | 房主→全体 | | **逐 tick 行情推送（本游戏核心新增频道）** |
| `act` | 玩家→房主 | | 交易/贷款/道具等操作请求 |
| `st/<pid>` | 房主→某玩家 | ✓ | 该玩家私有资产快照（现金/持仓/负债/净值） |
| `rank` / `spec` | 房主→全体 | ✓ | 公共实时排行榜 + 观战行情快照 |
| `fx` | 房主→全体 | | 动画特效（成交、爆仓、道具） |
| `chat` | 全体 | | 聊天 / 快捷喊话 |

### 6.3 房主权威要点

- **房主独占**：行情时钟、随机种子、AI、所有人的现金/持仓/负债、手续费/利息/结算逻辑。
- 真人玩家把操作发到 `act`，房主校验（余额、是否欠债禁贷、保证金、是否在当前 tick）后执行，再把私有资产快照发回 `st/<pid>`、把公共排行发到 `rank`。
- **反作弊（关键设计）**：行情采用**房主逐 tick 推送（`feed`）**，未来的价格序列**只存在房主端**，客户端永远只收到"已揭示"的历史——这样即便客户端本地打包了数据，也无法预读未来。比单纯下发"种子让各端自算"更公平（后者客户端能开天眼）。
- 断线重连、retained 快照恢复、被踢检测（`wasIn && !inRoom` → 提示并 reload）等沿用 hearthStoneKill2 现成逻辑。

## 8. 道具模式（可选）

- 房间勾选**道具模式**后启用；`propIntervalSec` 默认 60 秒发一个，可调。
- 每隔间隔，房主给每位在场玩家（含 AI）**随机发放一个小道具**到道具栏（栏位上限默认 3，满则不再发或顶替最旧）。
- 道具分**自用增益 / 信息 / 搅局(PvP)** 三类，点击使用，部分需选目标。
- 全部道具清单与数值见 **`道具图鉴.md`**。示例：内幕消息（看未来几 tick 方向）、黑天鹅（砸盘）、免单券（本笔免手续费）、止损盾（免一次爆仓）、加息冲击（抬高对手贷款利率）、冻结（让对手数秒不能交易）、抄底券（双倍杠杆一次）等。

## 9. 技术选型与目录结构

### 9.1 选型（与 hearthStoneKill2 同栈，降低复用成本）

- **构建**：Vite + 原生 JS（ES Modules），零框架。
- **联机**：`mqtt` npm 包（公共 broker），直接复用 `MqttBus`。
- **K 线图**：优先 TradingView `lightweight-charts`（MIT、轻量、漂亮）；或自绘 Canvas 蜡烛图（零依赖、完全可控、纯离线）。
- **数据**：构建期把各市场历史 K 线预处理成紧凑 JSON 打包进 `src/engine/data/`，运行时离线可用。

### 9.2 建议目录

```
StockClash/
├─ index.html
├─ package.json          # 依赖: mqtt, (lightweight-charts); dev: vite
├─ vite.config.js
├─ src/
```

## 10. 数据与法务备注

- 历史行情仅作**游戏化娱乐**用途；费率/利率均为可调的**平衡数值**，非真实交易参数，不构成任何投资建议。
- 数据获取：构建期离线抓取/导入公开历史 K 线并预处理打包（注意各数据源使用条款），运行时不联网拉取行情，避免 CORS 与可用性问题。
- 标的池建议覆盖牛市、熊市、震荡、暴涨暴跌等多种区间，保证重玩体验与公平。

## 11. 开发里程碑

| 阶段 | 目标 | 交付 |
| --- | --- | --- |
| **M1 单机 MVP** | 行情回放 + 买卖 + 手续费 + 结算 + 排行（1 真人 vs AI） | 能跑完整一局 |
| **M2 经济深化** | 做多做空 + 保证金/爆仓 + 贷款(放款即计+30s复利) + 主动还款 | 完整经济系统 |
| **M3 多市场** | 多市场数据集 + K线来源多选 + 随机开局 + 盲盒 | 重玩性 |
| **M4 联机** | MQTT 房主权威 + 房间管理(踢人/换位/增减AI/调难度) + 断线重连 | 可联机开黑 |
| **M5 道具模式** | 道具系统 + 道具栏 + PvP 道具 + AI 会用道具 | 见 `道具图鉴.md` |
| **M6 打磨** | Claude 美术设计、动效、音效、平衡性调参、手机适配 | 上线版 |

## 12. 待定/可讨论项

- 是否允许**多空同时持有**（对冲）还是单方向持仓？默认：同一时刻只持一个方向，反向操作先平后开。
- 破产玩家是否可**观战**剩余对局（建议：可，沿用观战席）。
- 道具是否对 **AI 也生效 / AI 是否会针对真人放搅局道具**（建议：是，更有对抗感）。
- 盲盒模式默认开还是关（建议：联机默认开，单机默认关）。
- 排行并列时的次级排序指标（建议：最大回撤更小者优先）。
